AngularJS Routing with Custom Data

This experiment demonstrates setting static custom data in the route configuration for differnt routes in AngularJS.

Demo:

Page 1 Page 2

Code snippet:

HTML: <nav> <a href="#/page1">Page 1</a> <a href="#/page2">Page 2</a> </nav> <div ng-view> </div> Javascript Routing: var app = angular.module('AngularRoutingCustomDataApp', ['ngRoute']); app.config(['$routeProvider', function ($routeProvider) { $routeProvider. when('/page1', { templateUrl: 'partials/page1.html', controller: 'Page1Controller', footer: 'This is Page 1 footer' }). when('/page2', { templateUrl: 'partials/page2.html', controller: 'Page2Controller', footer: 'This is Page 2 footer' }). otherwise({ redirectTo : '/page1' }); }]); app.controller("Page1Controller", function ($scope, $route) { $scope.footer = $route.current.footer; }); app.controller("Page2Controller", function ($scope, $route) { $scope.footer = $route.current.footer; });

As you can see in the route configuration, a parameter 'footer' is set for routes for Page 1 and 2. These parameters are currently hardcoded but we can easily have them dynamically assigned. The parameters can be accessed in the respective controller as shown in the above snippet
