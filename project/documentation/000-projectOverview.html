Project Overview

BeingChef is all about solving the 'What should we make tonight?' problem for you. It enables enthusiastic cooks to experiment different cuisines, save favorite recipes for future reference, provide productive comments on recipes, for example: providing an alternative ingredient. It also provides users an easy way to fill their grocery lists with the ingredients of a paricular recipe that they are viewing.

Additionally, one of the important motivations behind building this web application is to address a section of population, especially students, that move away from home and are burdened with the responsibility to cook for themselves. BeingChef is designed to address that burden.

Brief description of the web pages designed as part of the project:

  • Home Page: This is the landing page of the web application. A user can starting searching for recipes from this page.
  • Search Result/Search Page: Searching recipes on the home page will redirect users to this page and will display recipe results as tiles. The user can also perform additional searches from this page and need not revist the home page
  • Recipe Details Page: The user will be navigated to this page on trying to view the details of a particular recipe. On this page a user can add the recipe to it's favorite's list, add rating/comment for the recipe or add the recipe ingredients to it's grocery list.
  • User Dashboard: This page provides navigation tabs to view the following information:
    1. Favorite Recipes
    2. Recipe Reviews
    3. People Following
    4. Grocery List Management
  • Login/Register: The login and register functionality can be viewed in the header navigation bar of the application. In order to perform any user specific actions, like marking a recipe as favorite or leaving a review or a comment the user must login or register itself.
  • User public page: This page can be be accessed from either the user links in the recipe review section or the my searching a particular user from the dashboard with the 'Following' tab.

Technologies used as part of the project:

  • AngularJS
  • NodeJS
  • MongoDB and Mongoose
  • jQuery
  • Styling using Boostrap
