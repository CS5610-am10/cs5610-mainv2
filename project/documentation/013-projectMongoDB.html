BeingChef MongoDB Schema

The back end of the BeingChef application is MongoDB where recipe and user information is managed in various collections. The Mongoose API is used in the NodeJS to interact with the database.

User authentication is managed via the PassportJS local strategy along with Express' session management.

Below is the back end schema description of the application:

  • User Schema attributes:

    1. username: String
    2. password: { type: String, select: false } selection of password disabled
    3. firstName: String
    4. email: String
    5. favorites: [FavoriteSchema]
    6. reviews: [Schema.ObjectId]
    7. following: [Schema.ObjectId]
    8. groceryList: [IngredientSchema]

  • Recipe Schema attributes:

    1. title: String
    2. externalId : String
    3. description: String
    4. url: String
    5. reviews: [Schema.ObjectId]

  • Recipe Schema attributes:

    1. rating: {type: Number, default: 0, max: 5}
    2. comment: String
    3. creationDate: {type:Date, default:Date.now}
    4. username: String
    5. recipeTitle: String
    6. recipeUrl: String
    7. recipeId: String

  • Ingredient Schema attributes:

    1. name: String
    2. quantity: Number
    3. unit: String

  • Favorites Schema attributes:

    1. recipeTitle: String
    2. recipeUrl: String
    3. recipeId: String
